Lab: Deploying A Flink Application

  1. In the nfjs-flink-full-day application, go to the docker-compose.yaml. Right-Click and select Docker Compose Up.

  2. View the docker-compose.yaml and view the component parts. In particular, the job-manager and the task-manager.

  3. Open the basic-stream module, and view the com.evolutionnext.HighEndOrdersStream. View the code that we are running and notice the distinct parts of the Flink application

    1. The environment

    2. The source

    3. The sink

    4. The filter

    5. The execution

  4. Open a terminal

  5. Create a couple of topics in Kafka that we can use as a source and the sink

    docker exec broker kafka-topics --create --topic my_orders --bootstrap-server localhost:9092 --partitions 3 --replication-factor 1
docker exec broker kafka-topics --create --topic my_high_end_orders --bootstrap-server localhost:9092 --partitions 3 --replication-factor 1

  6. Ensure that the topics have been created

    docker exec broker kafka-topics --list --bootstrap-server localhost:9092

  7. Open a terminal and in the basic-stream module, run mvn package, and notice the target/basic-stream-1.0.jar. This is an uber jar, with all the dependencies that it needs.

  8. Go to the kafka-producer module, in this module we are going to run a kafka-producer that produces messages to the topic my_orders. You can start the producer in either the IDE itself, or you can use maven.

    mvn compile exec:java -Dexec.mainClass=com.evolutionnext.MyProducer

  9. Go back to the basic-stream module, and let’s submit the basic-stream-1.0.jar file to the job manager.

  10. Create a flink container with the executable, and run it so your job runs

    docker run -it \
  --network basic-stream_default \
  -v $(pwd)/target:/target \
  apache/flink:1.20.1-scala_2.12-java11 \
  /bin/bash

  11. cd into the target folder, and submit the job

    flink run --jobmanager http://flink-jobmanager:8081 basic-stream-1.0.jar

  12. Go to the Containers menu, and right-click the job manager. Select "Open in Browser". Here you should see the Flink UI. The Flink UI is your view of the job that is running. You should see your job currently running in the "Running Jobs" menu. Discover the UI, by looking around.

  13. Let us verify that the stream is working by viewing the filtered topic. We first look at the "before topic"

    docker exec -it broker kafka-console-consumer \
    --topic my_orders \
    --bootstrap-server broker:9092 \
    --property print.key=true \
    --from-beginning \
    --property key.separator=" : " \
    --property key.deserializer=org.apache.kafka.common.serialization.StringDeserializer \
    --property value.deserializer=org.apache.kafka.common.serialization.IntegerDeserializer

  14. Now, we look at the after topic

    docker exec -it broker kafka-console-consumer \
    --topic my_high_end_orders \
    --bootstrap-server broker:9092 \
    --property print.key=true \
    --from-beginning \
    --property key.separator=" : " \
    --property key.deserializer=org.apache.kafka.common.serialization.StringDeserializer \
    --property value.deserializer=org.apache.kafka.common.serialization.IntegerDeserializer

  15. Go to the UI, and cancel the job

View the Postgres Write Database

  1. Login into your postgres-write container using Attach Shell or docker exec -it postgres /bin/bash

  2. Run the following: export PGPASSWORD='docker'

  3. Run the following: psql -d docker -U docker

  4. In the Postgres shell run \dt which will show all the tables

  5. In the Postgres shell run \d stock_trade, which will show specific table schema

  6. Run SELECT * from stock_trade; and ensure that the data exists

  7. Exit the postgres container by CTRL+D and typing exit in the shell

Create the Postgres Connector

  1. Log into the Confluent Control Center

  2. Select your cluster controlcenter.cluster

  3. Select Connect in the menu

  4. Select the connect_default cluster

  5. Select the Add Connector button

  6. Select the JdbcSourceConnector button

  7. Add the following in the respective fields:

    1. Key Converter Class - io.confluent.connect.avro.AvroConverter

    2. Value Converter Class - io.confluent.connect.avro.AvroConverter

    3. JDBC URL - jdbc:postgresql://postgres-write:5432/

    4. JDBC User - docker

    5. JDBC Password - docker

    6. Database Dialect PostgreSqlDatabaseDialect

    7. Table Loading Mode incrementing

    8. Topic Prefix - postgres_

    9. Additional Properties - key.converter.schema.registry.url set to http://schema-registry:8081

    10. Additional Properties - value.converter.schema.registry.url set to http://schema-registry:8081

  8. Click Next

  9. Verify the JSON output, then select Launch

  10. Go back to the home page of the Confluent Control Center

  11. Go to the topics, and select postgres_stock_trade

  12. Select the Messages menu

  13. View the data coming for data loading

  14. You can stop the database loading by initiating CTRL+C

Run Flink SQL

  1. Login into flink-jobmanager container by using either Attach Shell on Gitpod or docker exec -it connect /bin/bash

  2. Run FlinkSQL

    sql-client.sh embedded

  3. Create a connector to Kafka

    CREATE TABLE stock_trades (
    id INT,
    stock_symbol STRING,
    trade_timestamp TIMESTAMP(3),
    trade_type STRING,
    amount DECIMAL(10, 2)
) WITH (
    'connector' = 'kafka',
    'topic' = 'postgres_stock_trade',
    'properties.bootstrap.servers' = 'broker:29092',
    'properties.group.id' = 'flink-sql-consumer',
    'scan.startup.mode' = 'earliest-offset',
    'format' = 'avro-confluent',
    'avro-confluent.schema-registry.url' = 'http://schema-registry:8081'
);

  4. Verify that FlinkSQL is working well by running a query

    SELECT * from stock_trades;

  5. Create a table in postgres for writes

    CREATE TABLE stock_count (
    stock_symbol STRING NOT NULL,
    trade_count BIGINT NOT NULL,
    last_updated TIMESTAMP(3) NOT NULL,
    PRIMARY KEY (stock_symbol) NOT ENFORCED
) WITH (
    'connector' = 'jdbc',
    'url' = 'jdbc:postgresql://postgres-read:5432/docker',
    'table-name' = 'stock_count',
    'username' = 'docker',
    'password' = 'docker',
    'sink.buffer-flush.max-rows' = '100',
    'sink.buffer-flush.interval' = '1s',
    'sink.max-retries' = '3'
);
    INSERT INTO stock_count
SELECT
    stock_symbol,
    COUNT(*) AS trade_count,
    CURRENT_TIMESTAMP AS last_updated
FROM stock_trades
GROUP BY stock_symbol;
